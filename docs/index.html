An open-source edge gateway that reads OPC UA and Modbus data and publishes it to MQTT Brokers — configured entirely through a web UI.

View on GitHub Quick Start

How it works

The gateway bridges industrial field devices with cloud platforms using open standards.

Features

Everything you need to connect factory-floor equipment to the cloud.

OPC UA & Modbus Inputs

Connect to any OPC UA server or Modbus TCP device. Configure polling, subscriptions, and register mapping from the UI.

OPC UA Node Browser

Visually navigate the OPC UA address space, inspect node attributes, and add tags to your acquisition list with bulk-select support.

Web UI Configuration

Auto-save configuration for all inputs and outputs. No command-line editing of config files required.

Telegraf-powered

Uses InfluxData Telegraf as the data engine. The UI generates and deploys telegraf.conf automatically.

Live Dashboard

Real-time pipeline health metrics — OPC UA read counts, MQTT publish rates, CPU, memory, and disk.

Docker Native

Designed to run with docker compose up. Includes demo OPC UA server and MQTT broker for local testing.

See it in action

A complete web UI to configure, monitor, and operate your edge gateway.

Pipeline Dashboard — live pipeline metrics, system health

Quick Start

Up and running in under two minutes.

  1. 1

    Clone the repository

    git clone https://github.com/cantalejo84/iiot-edge-gateway.git cd iiot-edge-gateway
  2. 2

    Start the full stack

    Includes the gateway, Telegraf, a demo OPC UA server, and a Mosquitto broker.

    docker compose up --build -d
  3. 3

    Open the web UI

    Configure your OPC UA / Modbus inputs and MQTT output, then click Deploy.

    http://localhost:8050
